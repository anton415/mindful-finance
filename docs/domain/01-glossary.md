# ТЗ-06. Глоссарий финансовой модели

**Статус:** рабочий проект  
**Назначение документа:** однозначно определить ключевые термины финансовой модели Mindful Finance v1 и исключить дублирование смыслов между product-, domain- и calculation-документами.  
**Связь с Техническим заданием:** материал используется при подготовке терминологической базы ТЗ, схемы данных, API-описаний и бизнес-правил.  
**Источник в репозитории:** `docs/domain/01-glossary.md`

## 1. Назначение словаря

Настоящий словарь фиксирует рабочие определения терминов, используемых в финансовой модели Mindful Finance v1. Каждый термин должен иметь одно доменное значение; если в продуктовой речи возможны синонимы, они приводятся только как допустимые алиасы, но не как отдельные сущности.

## 2. Основные термины

| Термин | Рабочее определение | Допустимые алиасы | Комментарий |
| --- | --- | --- | --- |
| `account` | Счет учета, в рамках которого фиксируются финансовые факты в одной расчетной валюте | счет учета | Не тождественен портфелю |
| `transaction` | Первичный факт финансового события, влияющий на ledger truth | операция учета, учетная операция | Не равен derived metric |
| `asset` | Финансовый инструмент, к которому может относиться позиция или дивиденд | инструмент, актив | Справочная сущность |
| `price snapshot` | Наблюдаемая цена актива на дату `asOf` из явного источника | ценовой снимок, котировка на дату | Используется только для valuation |
| `cash` | Денежная часть портфельного состояния, выраженная по валютам | кэш, денежный остаток | Не включает стоимость позиций |
| `cash balance` | Расчетный остаток денежных средств в одной валюте на дату `asOf` | остаток кэша | Derived state |
| `position` | Расчетное владение определенным активом в выбранном scope | позиция по активу | Derived state, а не первичный факт |
| `portfolio` | Логический aggregate scope над выбранным набором счетов | портфель | В `v1` не является первичным mutable truth |
| `portfolio state` | Полное расчетное состояние портфеля на дату `asOf`: cash balances, positions и сопутствующие derived values | состояние портфеля | Строится только из фактов и valuation inputs |
| `asset value` | Стоимость позиции по одному активу на дату `asOf`, рассчитанная как `quantity * price snapshot` | стоимость актива | Без snapshot не вычисляется |
| `portfolio value` | Совокупная стоимость кэша и позиций на дату `asOf`, по валютным корзинам или в расчетной валюте при явном FX input | стоимость портфеля | Не хранится как первичный факт |
| `contribution` | Пополнение портфельного контура внешними деньгами | пополнение | Увеличивает funding, но не investment return |
| `withdrawal` | Вывод денег из портфельного контура наружу | вывод | Уменьшает funding; в расходы попадает только при явной расходной квалификации |
| `dividend` | Денежное распределение дохода по активу | дивиденд | По умолчанию увеличивает cash и считается investment return |
| `fee` | Явная комиссия или издержка операции | комиссия | Никогда не должна теряться в "грязной" сумме |
| `annual expenses` | Годовая база расходов, используемая для FIRE-расчетов | annual spending, annual burn basis | Требует явного указания basis |
| `realized_annual_expenses_ttm` | Derived metric расходов за trailing 12 months на основе fact-операций, признанных расходными | фактические annual expenses TTM | Не равен planning estimate |
| `planning_annual_expenses` | Явная оценка будущих годовых расходов для planning/FIRE | planning expenses, target annual expenses | Estimate input |
| `safe withdrawal rate` | Явный planning-параметр, показывающий допустимую долю вывода портфеля в год для расчета FIRE target | `SWR`, safe rate | В домене хранится как ratio, не как UI-строка |
| `FIRE goal` | Planning-цель финансовой независимости, связывающая scope, annual expenses и safe withdrawal rate | цель FIRE | Не тождественна derived progress |
| `progress to FIRE` | Derived metric прогресса к FIRE, рассчитываемый как `investable_portfolio_value / fire_target` | FIRE progress | Raw ratio не ограничивается 100% |
| `fact` | Наблюдаемый или зафиксированный учетный факт, который относится к truth-слою системы | truth record, факт учета | Не зависит от сценариев и предположений |
| `plan` | Намерение пользователя совершить контролируемое будущее действие | planned event, план | Не является truth до фактического наступления |
| `scenario` | Гипотетический набор событий и параметров для моделирования "что если" | what-if scenario, сценарий | Не изменяет truth-ledger |
| `estimate` | Оценочное числовое предположение или параметр модели | assumption, estimate | Должно иметь явный источник и область применения |

## 3. Правила употребления терминов

В `v1` действуют следующие правила употребления:

1. термины `portfolio` и `portfolio state` не взаимозаменяемы: первый означает scope-агрегат, второй - состояние этого агрегата на дату `asOf`;
2. термин `annual expenses` не должен использоваться без указания basis, если документ или код различают `realized_annual_expenses_ttm` и `planning_annual_expenses`;
3. термин `portfolio value` обозначает результат вычисления, а не сохраненное произвольное число;
4. термин `dividend reinvestment` обозначает составное business-event, а не отдельный неразложимый transaction type;
5. термин `estimate` не допускается как эвфемизм для факта, а термин `scenario` не допускается как эвфемизм для плана.

## 4. Анти-синонимы и недопустимые смешения

Следующие смысловые смешения считаются недопустимыми:

- `portfolio value` нельзя использовать как синоним `net worth`, если часть стоимости находится вне выбранного portfolio scope;
- `withdrawal` нельзя автоматически считать `expense`, пока не задана явная расходная квалификация;
- `price snapshot` нельзя использовать как синоним future return assumption;
- `safe withdrawal rate` нельзя трактовать как инвестиционную рекомендацию или юридическое обещание;
- `fact`, `plan`, `scenario` и `estimate` нельзя хранить в одном logical bucket без явной маркировки.
