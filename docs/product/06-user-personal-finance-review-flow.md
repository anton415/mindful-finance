# Guide-User. Спокойный review личных финансов

**Статус:** рабочий документ  
**Назначение документа:** описать базовый пользовательский сценарий спокойного review в разделе личных финансов.  
**Связь с Техническим заданием:** документ конкретизирует практический пользовательский flow внутри milestone `Учет личных финансов и спокойный review`.  
**Источник в репозитории:** `docs/product/06-user-personal-finance-review-flow.md`

## 1. Назначение сценария

Сценарий review нужен для того, чтобы пользователь за один сеанс получил целостную картину личных финансов без ручных обходов и без перегруженного интерфейса. Результатом review должен быть ответ на вопросы: какие фактические расходы и доходы уже зафиксированы, как меняется баланс по картам, и какое следующее действие требуется.

## 2. Подготовка к review

Перед началом review рекомендуется:

1. выбрать нужный год в разделе личных финансов;
2. убедиться, что рабочая карта находится в статусе `ACTIVE`;
3. проверить, что в карточке есть базовые настройки (baseline, лимиты, доходный прогноз), если они используются в текущем месяце.

## 3. Основной цикл review

### 3.1. Вкладка расходов

Во вкладке расходов пользователь фиксирует фактические суммы по категориям. Годовой обзор агрегируется по активным картам, а факт сохраняется в выбранную карту и влияет на текущий баланс.

### 3.2. Вкладка доходов

Во вкладке доходов пользователь сверяет помесячные суммы и статусы (`FACT`, `FORECAST`, `OVERRIDE`) для понимания, какие месяцы закрыты фактом, а какие рассчитываются прогнозом.

### 3.3. Вкладка факт/план доходов

На полной странице факт/план пользователь может внести фактический доход за месяц и настроить planner отпусков или 13-й зарплаты. Фактическое значение имеет приоритет над прогнозным.

### 3.4. Вкладка настроек карты

Во вкладке настроек пользователь может переименовать карту, обновить параметры лимитов и дохода, а также управлять жизненным циклом карты (archive/restore/delete) в рамках доступных ограничений.

## 4. Перевод между картами

Перевод между картами используется для перемещения суммы между двумя активными картами без искажения фактических расходов. Корректный перевод:

- создается между разными картами;
- выполняется с положительной суммой;
- требует даты операции.

## 5. Ожидаемый результат review-сеанса

Сеанс review считается завершенным, когда пользователь:

- видит актуальный баланс по рабочим картам;
- понимает годовые и помесячные расходы/доходы;
- может назвать одно следующее действие (например, скорректировать лимит, обновить факт дохода или выполнить перевод).

## 6. Базовые проверки при расхождениях

Если значения выглядят некорректно, рекомендуется:

1. проверить выбранный год и выбранную карту;
2. убедиться, что карта не находится в архивном статусе;
3. повторно открыть раздел после сохранения фактов или перевода;
4. обратиться к разработчику с указанием карты, месяца и шага, где возникло расхождение.
