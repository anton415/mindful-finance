# Материалы к Техническому заданию Mindful Finance

Данный каталог открывает исходный набор документов, используемых при подготовке Технического задания на разработку приложения Mindful Finance. Локальные markdown-файлы считаются редактируемым исходником и должны синхронизироваться с одноименными страницами в GitHub Wiki, где страница `Home` выступает глобальным индексом материалов `ТЗ-00...09`.

## Состав материалов ТЗ

1. [ТЗ-00. Контекст продукта](./00-context.md) - назначение продукта, целевая аудитория, проблема, ценность, принципы `Calm/Truth` и ограничения первой версии.
2. [ТЗ-01. Продуктовая концепция и ценность](./01-product-concept-and-value.md) - продуктовая концепция, целевая аудитория, ценностное предложение и базовые продуктовые принципы.
3. [ТЗ-02. Границы версии v1](./02-v1-scope.md) - состав первой версии, критерии готовности, допущения и релизные ограничения.
4. [ТЗ-03. Основные пользовательские сценарии v1](./03-v1-use-cases.md) - ключевые пользовательские задачи, определяющие практическую ценность `v1 / closed beta`.
5. [ТЗ-04. Ограничения и исключения v1](./04-v1-exclusions.md) - перечень инициатив, сознательно исключаемых из первой версии для защиты от feature creep.
6. [ТЗ-05. Финансовая доменная модель v1](../domain/00-domain-draft.md) - состав сущностей, связи, слои `truth / valuation / planning` и границы финансовой модели.
7. [ТЗ-06. Глоссарий финансовой модели](../domain/01-glossary.md) - однозначные определения терминов финансовой модели и допустимых алиасов.
8. [ТЗ-07. Доменные правила и ограничения](../domain/02-business-rules.md) - инварианты, типы операций и правила их влияния на состояние системы.
9. [ТЗ-08. Правила расчётов портфеля и FIRE](../domain/03-calculation-rules.md) - пошаговые правила расчета `portfolio state`, `portfolio value`, annual expenses basis и `progress to FIRE`.
10. [ТЗ-09. Определения `fact / plan / scenario / estimate`](../domain/04-model-definitions.md) - понятийные границы между truth-, planning-, scenario- и estimate-слоями.

## Структура материалов

Материалы ТЗ в `v1` делятся на два связанных блока:

- `ТЗ-00...04` - product framing: контекст, ценность, границы версии, пользовательские сценарии и исключения;
- `ТЗ-05...09` - domain framing: финансовая модель, словарь терминов, бизнес-правила, правила расчетов и понятийные границы между truth/planning/scenario слоями.

Такое деление нужно для того, чтобы product-уровень не подменял собой формальную финтех-логику, а domain-уровень можно было напрямую использовать как исходник для schema v1 и business logic.

## Правила синхронизации

- GitHub Wiki используется как навигационный слой для всех материалов ТЗ, а не только для product-блока.
- Изменения в содержании следует сначала вносить в локальные файлы `docs/product/` и `docs/domain/`, а затем зеркалировать их в wiki.
- Структура документов выполнена в адаптированном стиле ГОСТ 34.602-89 для markdown и wiki: формальный язык, явное назначение документа и нумерованные разделы.
- Для внутренних alias-ссылок в GitHub Wiki следует использовать синтаксис `[[Текст ссылки|Имя страницы]]`; левая часть служит человекочитаемой подписью, правая - точным именем wiki-страницы.
- Если человекочитаемая подпись содержит символы, неудобные для имени страницы, следует упрощать именно имя страницы, сохраняя полную подпись в левой части alias-ссылки.
